# MSGSNet

MSGSNet is a sequence-graph dual-branch collaborative modeling framework for transcription factor binding site (TFBS) prediction that models DNA sequences from complementary semantic and structural viewpoints. 

![image](https://github.com/yamadaanna111/MSGSNet/blob/main/model.jpg)

---

## Key Features

- Hybrid sequence–graph framework for transcription factor binding site prediction.  
- Sequence semantic module: multi-scale convolutions and Bi-LSTM for local and long-range dependency modeling, enhanced by hierarchical bidirectional attention (HBA).  
- Multi-scale graph module: De Bruijn graphs at different k-mer scales processed by ChebConv and graph channel attention (GCA), with paired cross-attention for scale-aware fusion.  
- Dual-branch fusion: integration of sequence and graph representations via bidirectional cross-attention, followed by MLP-based classification.

## Environment

- Python 3.10.13  
- PyTorch 2.1.2  
- numpy 1.26.3  
- scikit-learn 1.3.2  
- pandas 2.1.4  

An `environment.yml` file is provided to facilitate environment setup.

```bash
conda env create -f environment.yml
conda activate msgsnet
```

## Dataset

The benchmark datasets used in this study are derived from **165 ENCODE ChIP-seq datasets** curated by *Zeng et al.*, covering **29 transcription factors across 32 cell lines**.

- **Sequence length**: 101 bp
- **Positive samples**: DNA fragments centered on ChIP-seq peaks, experimentally validated to contain at least one TFBS
- **Negative samples**: sequences generated by nucleotide-preserving random shuffling of positive samples
- **Class balance**: equal numbers of positive and negative samples

Due to data usage policies, users are expected to prepare the datasets following the described protocol or refer to the original ENCODE resources.

## Training and Evaluation

Model training and evaluation can be performed using:

```bash
python train.py -e 300
```

- `-e`: number of training epochs

The best-performing model parameters (`best_model.pt`) and evaluation results are saved under the `experiments/` directory.
